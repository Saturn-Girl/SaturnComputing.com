This page contains the development logs, personal history, and technical details of YanderePython.exe.

A proprietary project originally made in Ren’Py 7.5 using x86 assembly and Python. Developed by a coder with a massive arsenal of languages including PHP, C++, Assembly, and Rust. A fan of old tech, GTA, and circuit wiring.

Development Timeline

The project began around Halloween 2024. Originally named Yandere School Python Simulator, it was later renamed YanderePy.

Fun Fact: The game originally used raw OpenGL before moving to full game engines.

By late 2024, the plan shifted to Unity. Despite learning C# from Mosh, the early Unity versions only featured a falling cube. Eventually, the project switched to Godot 3.2 using GDScript.

2025 Development & Engines

In early 2025, the project experimented with Pygame + PyOpenGL. March saw the release of the Ren'Py Pocket Edition Prototype—the first proprietary game from Saturn Computing.

Licensing: The project uses a proprietary license with a strict EULA. Windows 95–98 support was eventually abandoned in favor of Windows 7 compatibility (Target size: 1.2 GB).

Art & Tools

Visuals were crafted in MS Paint and GIMP. 3D work happened in Blender 2.80 (Light Theme) and 3DS MAX. The project briefly moved to the Source Engine in July 2025 before being ported to Raylib for performance.

Ren’Py & Release Cycle

September 2025 was chaotic. The developer switched between Unity and Ren’Py multiple times. The game was scrapped on September 6 but revived shortly after. YanderePython.exe finally released on September 17, with Episode 1 arriving September 26, 2025.

Fun Fact: The character Yuki uses modified assets from the 2016 Yandere Simulator build.

Blender Game Engine / YanderePython.exe - 2

While testing UPBGE, the developer started YanderePython.exe - 2. Despite using a GPL engine, this project remains proprietary.

YanderePython.exe Impact - 2 (GameMaker)

While we don't want to spoil too much, Impact - 2 runs on GameMaker. The game is set in 2028 with an older protagonist. It is planned to have a very sad ending set in the 2030s.

About the Developer

The developer is a 12-year-old programmer (7th Grade) who mastered elementary school in April 2025. He uses 6502 and x86 Assembly and works primarily on Windows after various struggles with Linux GRUB bootloaders.

He is silent, chill, has autism, and occasionally ignores online classes to finish his code. "Ain’t no way he is a Project Sekai fanboy..."
